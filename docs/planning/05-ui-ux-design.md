## Animation / Transitions

### Animation Philosophy
- **Purposeful**: Animations aid understanding, never just decorative
- **Subtle**: Quick and unobtrusive - users shouldn't wait for animations
- **Consistent**: Same duration/easing for similar actions
- **Reducible**: Respect `prefers-reduced-motion` OS setting

### Timing Tokens
| Speed | Duration | Use Case |
|-------|----------|----------|
| Instant | 0ms | Immediate feedback (button press) |
| Fast | 100ms | Hover states, small transitions |
| Normal | 200ms | Panel switches, tab changes |
| Slow | 300ms | Modal open/close, page transitions |

### Easing
- **Default**: `ease-out` (cubic-bezier(0, 0, 0.2, 1)) - quick start, gentle stop
- **Enter**: `ease-out` - elements entering feel responsive
- **Exit**: `ease-in` (cubic-bezier(0.4, 0, 1, 1)) - elements leaving accelerate away
- **Bounce**: Reserved for success/celebration states only

### Page/View Transitions

**Tab Switch**
- Duration: 150ms
- Effect: Instant content swap (no sliding)
- New content fades in

**Right Panel Mode Switch (Preview/Activity/Dashboard)**
- Duration: 200ms
- Effect: Crossfade between views

**Full Project View Open**
- Duration: 300ms
- Effect: Right panel expands to full width, chat slides left and fades
- Reverse on close

**Settings/Session Browser Open**
- Duration: 250ms
- Effect: Slide in from right + fade

**Modal Open**
- Duration: 200ms
- Effect: Fade in + scale from 95% to 100%
- Backdrop fades in

### Loading States

- No skeleton/spinner - content streams in naturally
